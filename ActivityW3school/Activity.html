HTML HEADING

HTML headings are titles or subtitles that you want to display on a webpage.

Here's a tutorial on how to use HTML style:

  1. Basic Structure:
    2. Headings are defined with the <h1>to <h6> tags.
  2. Semantic Meaning:
    3. Choose the appropriate heading level based on the importance of the content. <h2> represents the subsections, <h3> for sub-subsections, and so on. This hierarchy should make sense structurally and semantically.
  3. Formatting:
    4. By default, web browsers apply different formatting to each heading level. The formatting may include larger fonts and bold text. However, you can modify the appearance of headings with CSS to match your website's design.
  4. Accessibility:
    5. HTML headings are important for accessibility. Screen readers and assistive technologies rely on the hierarchy of headings to navigate and provide context to users with disabilities. Make sure your headings accurately represent the content structure.

HTML PARAGRAPH

HTML paragraphs are used to group and display text content in a block-level element. They are an essential part of structuring textual content on a web page.A paragraph always starts on a new line, and browsers automatically add some white space (a margin) before and after a paragraph.

Here's a tutorial on how to use HTML paragraphs:

  1. Basic Structure:
    2. HTML paragraphs are defined using the <p> element.
  2. Text Context:
    3. Place your text content within the <p> element. You can include plain text, links, inline formatting (e.g., <em> for italics, <strong> for bold), and other inline HTML elements.
  3. Block-level Element:
    4. The <p> element is a block-level element, which means it creates a block-level container for text. Paragraphs are typically separated from each other with vertical spacing.
  4. Whitespace:
    5. HTML collapses consecutive spaces and line breaks into a single space. If you want to preserve extra spaces or line breaks within a paragraph, you can use the <br> element for line breaks or the CSS white-space property to control the handling of spaces.
  5. Semantic Meaning:
    6. Use paragraphs to structure your text content logically. Each paragraph should represent a distinct idea or topic. Avoid using <p> elements solely for styling; use CSS for that purpose.
  6. Attributes:
    7. The <p> element doesn't have many attributes. However, you can use the <class> and <id> attributes to apply CSS styles or for JavaScript interaction.

HTML STYLE

Styling in HTML is typically accomplished using CSS (Cascading Style Sheets). CSS allows you to control the appearance and layout of HTML elements.

Here's a tutorial on how to use CSS to style your HTML documents:

  1. Creating a CSS to File:
    2. • Create a new file with a css extension (e.g., styles.css) to store your CSS rules.
    • Make sure this CSS file is in the same directory as your HTML file, or you can specify the correct file path when linking it in your HTML.
  2. Linking CSS to HTML:
    3. In your HTML document, link the CSS file in the <head> section using the <link> element.
  3. Adding CSS to HTML:
    4. In your styles.css file, you can define CSS rules to target and style specific HTML elements.
  4. Sectors and Properties:
    5. • Selector: CSS selectors define which HTML elements should be styled. In the example above, <p> selects all <p> elements, highlighted selects elements with the highlighted class, and #special-paragraph selects the element with the special-paragraph ID.
    • Properties: Properties define the style attributes you want to apply. Common properties include color (text color), font-size, background-color, font-weight, and many more.
  5. Cascading Nature:
    6. CSS is "cascading," which means that styles can be inherited from parent elements and overridden by more specific rules. The order of your CSS rules also matters; rules applied later in the CSS file take precedence.
  6. Testing and Refining
    7. After applying styles, it's essential to test your HTML document in different browsers to ensure that your styles are rendering correctly. You may need to refine your CSS rules to achieve the desired appearance.

HTML Formatting

HTML (Hypertext Markup Language) text formatting is the process that allows you to control the appearance of text within a web page. It includes elements and attributes that can be used to modify text style, such as font size, color, alignment, and more.

HTML Formatting Elements

Here are the formatting elements that were designed to display special types of text:

  • <b> - Bold text
  • <strong> - Important text
  • <i> - Italic text
  • <em> - Emphasized text
  • <mark> - Marked/highlighted text
  • <small> - Smaller text
  • <del> - Deleted text
  • <ins> - Inserted text
  • <sub> - Subscripted text
  • <sup> - Superscripted text

HTML <b> and <strong> Elements

  • The HTML <b> element defines bold text, without any extra importance.

    • Example:
             <b> This text is bold </b>

    Output:
              This text is bold

  • The HTML <strong> element defines text with strong importance. The content inside is typically displayed in bold.

    • Example:
              <strong> This text is important! </strong>

    Output:
              This text is important!

HTML <i>and <em> Elements

  • The HTML <i> element defines a part of text in an alternate voice or mood. The content inside is typically displayed in italic.

    • Example:
             <i> This text is italic </i>

    Output:
              This text is italic

  • The HTML <em> element defines emphasized text. The content inside is typically displayed in italic.
    Tip: A screen reader will pronounce the words in <em> with an emphasis, using verbal stress.

    • Example:
             <em> This text is emphasized </em>

    Output:
              This text is emphasized

HTML <mark> Element

  • The HTML <mark> element defines text that should be marked or highlighted:

    • Example:
             <p> Do not forget to buy <mark> flower </mark> today. </p>

    Output:
              Do not forget to buy flower today.

HTML <small> Element

  • The HTML <small> element defines smaller text:

    • Example:
             <small> This is some smaller text </small>

    Output:
              This is some smaller text

HTML <del> Element

  • The HTML <del> element defines text that has been deleted from a document. Browsers will usually strike a line through deleted text:

    • Example:
             <p> My favorite fruit is <del>apple</del> orange.</p>

    Output:
              My favorite fruit is apple orange.

HTML <ins> Element

  • The HTML <ins> element defines a text that has been inserted into a document. Browsers will usually underline the inserted text:

    • Example:
             <p> My favorite fruit is <del>apple</del> <ins>orange</ins>.</p>

    Output:
              My favorite fruit is apple orange.

HTML <sub> Element

  • The HTML <sub>element defines subscript text. Subscript text appears half a character below the normal line and is sometimes rendered in a smaller font. Subscript text can be used for chemical formulas, like H2O:

    • Example:
             <p>This is <sub>subscripted</sub> text</p>

    Output:
             This is subscripted text.

HTML <sup> Element

  • The HTML <sup> element defines superscript text. Superscript text appears half a character above the normal line and is sometimes rendered in a smaller font. Superscript text can be used for footnotes, like WWW [1]:

    • Example:
             <p>This is <sup> superscripted </sup> text.</p>

    Output:
             This is superscripted text.

HTML Quotations

In HTML, a "quotation" refers to a section of text that is quoted or attributed to a specific source. The primary purpose of HTML quotations is to indicate that a particular piece of text is a direct quote from another source, such as a book, article, or speech.

HTML Quotation and Citation Elements

  • <blockquote> - Defines a section that is quoted from another source
  • <q> - Defines a short inline quotation
  • <abbr> - Defines an abbreviation or acronym
  • <address> - Defines contact information for the author/owner of a document
  • <cite> - Defines the title of a work
  • <bdo> - Defines the text direction

HTML <blockquote> for Quotations

  • The HTML <blockquote> element defines a section that is quoted from another source.

    • Example:
             <p> Here is a quote about life: </p>
             <blockquote>
                 "Life is a journey that must be traveled no matter how bad the roads and accommodations. Sometimes, on this journey, you will find yourself lost in a dark forest, unsure of your path. At other times, you will bask in the warmth of the sun on an open meadow. Life is full of unexpected twists and turns, like a novel with pages still unwritten. Embrace every moment, both the trials and the triumphs, for they are the chapters of your story. In the end, it's not the destination that matters most but the experiences and lessons gained along the way."
             </blockquote>

    Output:

    Here is a quote about life:
    "Life is a journey that must be traveled no matter how bad the roads and accommodations. Sometimes, on this journey, you will find yourself lost in a dark forest, unsure of your path. At other times, you will bask in the warmth of the sun on an open meadow. Life is full of unexpected twists and turns, like a novel with pages still unwritten. Embrace every moment, both the trials and the triumphs, for they are the chapters of your story. In the end, it's not the destination that matters most but the experiences and lessons gained along the way."

HTML <q> for Short Quotations

  • The HTML <q> tag defines a short quotation.

    • Example:
             <p> A wise person once said, <q> Life is what you make it. </q></p>

    Output:
             A wise person once said, Life is what you make it.

HTML <abbr> for Abbreviations

  • The HTML <abbr> tag defines an abbreviation or an acronym, like "HTML," "CSS," "Mr.," "Dr.," "ASAP." Marking abbreviations can give useful information to browsers, translation systems, and search engines.

    • Example:
             <p> The <abbr title="United Nations Educational, Scientific and Cultural Organization"> UNESCO </abbr> celebrates life through art and culture.</p>

    Output:
             The UNESCO celebrates life through art and culture.

HTML <address> for Contact Information

  • The HTML <address> tag defines the contact information for the author/owner of a document or an article. The contact information can be an email address, URL, physical address, phone number, social media handle, etc. The text in the <address> element usually renders in italic, and browsers will always add a line break before and after the <address> element.

    • Example:
             <address>
                  Written by Jane Smith
                  Contact: <a href="mailto:jane@example.com">jane@example.com</a>
                  Visit us at: <a href="http://www.example.com">www.example.com</a>
                  Address: 123 Main Street, City, Country
             </address>

    Output:

    Written by Jane Smith
    Contact: jane@example.com
    Visit us at: www.example.com
    Address: 123 Main Street, City, Country

HTML <cite> for Work Title

  • The HTML <cite> tag defines the title of a creative work (e.g. a book, a poem, a song, a movie, a painting, a sculpture, etc.). Note: A person's name is not the title of a work.

    • Example:
             <cite> The Art of Living </cite> by John Doe. A book that explores the meaning of life.

    Output:
              The Art of Living by John Doe. A book that explores the meaning of life.

HTML <bdo> for Bi-Directional Override

  • BDO stands for Bi-Directional Override. The HTML <bdo> tag is used to override the current text direction.

    • Example:
             <bdo dir="rtl">This quote, <q>Life is a journey</q>, written in a right-to-left script.</bdo>

    Output:
              This quote, Life is a journey, written in a right-to-left script.

HTML Comments

HTML comments are non-rendered elements within an HTML document that serve the purpose of providing information, explanations, or notes about the HTML code. They are not visible to website visitors in the web browser but can be seen in the HTML source code. HTML comments are enclosed between <!-- and -->.

HTML Comment Tag

  • You can add comments to your HTML source code using the following syntax:

    • <!-- Write your comments here -->

    Note that there is an exclamation point (!) in the start tag, but not in the end tag.
    Note: Comments are not displayed in the browser, but they are useful for documenting your HTML source code.

Adding Comments

  • Comments can be used to include notifications and reminders within your HTML code:

    • Example:
         <!-- This is a comment -->
         <p> This is a paragraph. </p>

HTML Styles - CSS

CSS stands for Cascading Style Sheets, is a stylesheet language used for describing the presentation of a document written in HTML (Hypertext Markup Language) or XML (eXtensible Markup Language). CSS allows web designers and developers to control the layout, appearance, and formatting of web pages, making it an essential part of web design.

Here are some key points about CSS:

  1. Separation of Content and Presentation:
    2. CSS separates the content of a web page (HTML) from its visual presentation. This separation makes it easier to maintain and update websites since changes to the design can be made in the CSS without altering the underlying HTML content.

  2. Selectors and Properties:
    3. CSS uses selectors to target HTML elements and properties to define how those elements should be styled. For example, you can use a selector to target all the <p> elements in your HTML document and specify properties like font size, color, and margin to control their appearance.

  3. Cascading:
    4. The term "cascading" in CSS refers to the order of precedence when multiple styles are applied to the same element. There are three levels of CSS styles: user agent styles (default styles provided by the browser), external styles (defined in separate CSS files), and inline styles (defined directly in the HTML document). If conflicting styles are defined at different levels, the cascade rules determine which style takes precedence.

  4. External CSS
    5. It is common practice to define CSS styles in external CSS files with a .css extension. These files can be linked to HTML documents using the <link> element within the document's <head> section. This approach allows for consistent styling across multiple web pages.

  5. Inline CSS:
    6. You can also define CSS styles inline within HTML elements using the style attribute. Inline styles are applied directly to a specific element and take precedence over external styles.

  6. Internal CSS:
    7. CSS styles can be included directly within an HTML document using the <style> element within the document's <head> section. Internal CSS styles apply to the HTML document in which they are defined.

  • font-weight: Sets the boldness of the link text.

Link bookmarks, also known as anchors, are a way to create specific locations within a web page. You can create a bookmark by adding an id attribute to a specific element within the page, and then linking to that id using the href attribute of an <a> tag.

HTML Form Attributes

This chapter describes the different attributes for the HTML <form> element.

The Action Attribute <form action="">

The action attribute defines the action to be performed when the form is submitted. Usually, the form data is sent to a file on the server when the user clicks on the submit button.

The Target Attribute <form action="" target="_blank">

The target attribute specifies where to display the response that is received after submitting the form.

The target attribute can have one of the following values:

  • _blank The response is displayed in a new window or tab.
  • _self The response is displayed in the current window.
  • _parent The response is displayed in the parent frame.
  • _top The response is displayed in the full body of the window.
  • framename The response is displayed in a named iframe.

The Method Attribute <form action="" method="get">

The method attribute specifies the HTTP method to be used when submitting the form data. The form-data can be sent as URL variables (with method="get") or as HTTP post transaction (with method="post"). The default HTTP method when submitting form data is GET.

The Autocomplete Attribute <form action="" autocomplete="on">

The autocomplete attribute specifies whether a form should have autocomplete on or off. When autocomplete is on, the browser automatically complete values based on values that the user has entered before.

The Novalidate Attribute <form action="" novalidate>

The novalidate attribute is a boolean attribute. When present, it specifies that the form-data (input) should not be validated when submitted.

The input value attribute specifies an initial value for an input field.

The input size attribute specifies the visible width, in characters, of an input field.

The default value for size is 20.

Note: The size attribute works with the following input types: text, search, tel, url, email, and password.

The maxlength Attribute

The input maxlength attribute specifies the maximum number of characters allowed in an input field.

Note:When a maxlength is set, the input field will not accept more than the specified number of characters. However, this attribute does not provide any feedback. So, if you want to alert the user, you must write JavaScript code.

The min and max Attributes

The input min and max attributes specify the minimum and maximum values for an input field.

The min and max attributes work with the following input types: number, range, date, datetime-local, month, time and week.

Tip:Use the max and min attributes together to create a range of legal values.

The multiple Attribute

The input multiple attribute specifies that the user is allowed to enter more than one value in an input field.

The multiple attribute works with the following input types: email, and file.

The pattern Attribute

The input pattern attribute specifies a regular expression that the input field's value is checked against, when the form is submitted.

The pattern attribute works with the following input types: text, date, search, url, tel, email, and password.

Tip: Use the global title attribute to describe the pattern to help the user.

The placeholder Attribute

The input placeholder attribute specifies a short hint that describes the expected value of an input field (a sample value or a short description of the expected format).

The short hint is displayed in the input field before the user enters a value.

The placeholder attribute works with the following input types: text, search, url, tel, email, and password.

The required Attribute

The input required attribute specifies that an input field must be filled out before submitting the form.

The required attribute works with the following input types: text, search, url, tel, email, password, date pickers, number, checkbox, radio, and file.

The step Attribute

The input step attribute specifies the legal number intervals for an input field.

Example: if step="3", legal numbers could be -3, 0, 3, 6, etc.

Tip:Tip: This attribute can be used together with the max and min attributes to create a range of legal values.

The autofocus Attribute

The input autofocus attribute specifies that an input field should automatically get focus when the page loads.

The height and width Attributes

The input height and width attributes specify the height and width of an <input type="image"> element.

Tip:Always specify both the height and width attributes for images. If height and width are set, the space required for the image is reserved when the page is loaded. Without these attributes, the browser does not know the size of the image, and cannot reserve the appropriate space to it. The effect will be that the page layout will change during loading (while the images load).

The list Attribute

The input list attribute refers to a <datalist> element that contains pre-defined options for an <input> element.

The autocomplete Attribute

The input autocomplete attribute specifies whether a form or an input field should have autocomplete on or off.

Autocomplete allows the browser to predict the value. When a user starts to type in a field, the browser should display options to fill in the field, based on earlier typed values.

The autocomplete attribute works with <form> and the following <input> types: text, search, url, tel, email, password, datepickers, range, and color.
